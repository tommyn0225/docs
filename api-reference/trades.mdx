---
title: "GET /api/v1/trades/{token} — Fetch a Public Trade List"
sidebarTitle: "Trades"
description: "Retrieve a user's public trade list using their share token. Returns cards marked as available for trade. Returns 404 if the list is private or the token is invalid."
---

The **Fetch a Public Trade List** endpoint returns all cards a Binder user has marked as available for trade, identified by their unique share token. You can use this endpoint to build trade-matching tools, display a user's trade inventory on an external site, or power trade negotiation workflows.

<Note>
  Each user's trade list has its own share token separate from their binder IDs. You can find a user's trade share token in their profile URL on the `/trades` page. This endpoint requires no authentication.
</Note>

## Request

```http
GET https://yourdomain.com/api/v1/trades/{token}
```

### Path Parameters

<ParamField path="token" type="string" required>
  The trade list share token in UUID format. This token is generated by Binder and is unique to each user's trade list. You can find it in the share URL on the user's `/trades` page — for example, `https://yourdomain.com/trades/770e8400-e29b-41d4-a716-446655440002`.
</ParamField>

## Response

A successful request returns `200 OK` with a JSON object containing the share token and an array of holdings the user has marked as available for trade. Each holding includes normalized card metadata.

```json
{
  "token": "770e8400-e29b-41d4-a716-446655440002",
  "holdings": [
    {
      "id": "880e8400-e29b-41d4-a716-446655440003",
      "card_id": "pika001",
      "game": "pokemon",
      "quantity": 1,
      "card_data": {
        "name": "Charizard",
        "set": "base1",
        "rarity": "rare",
        "usd": "300.00",
        "image_uri": "https://images.pokewallet.io/..."
      }
    }
  ]
}
```

### Response Fields

<ResponseField name="token" type="string">
  The share token used to retrieve this trade list. Matches the `{token}` path parameter you provided.
</ResponseField>

<ResponseField name="holdings" type="array">
  An array of card holdings the user has marked as available for trade. Returns an empty array if the user has no cards marked for trade.

  <Expandable title="holdings[]">
    <ResponseField name="id" type="string">
      The UUID of the holding record within the user's binder.
    </ResponseField>

    <ResponseField name="card_id" type="string">
      The game-specific identifier for the card. The format varies by game — for example, MTG cards use Scryfall UUIDs while Pokémon cards use a compact identifier like `pika001`.
    </ResponseField>

    <ResponseField name="game" type="string">
      The trading card game this card belongs to. Possible values: `mtg`, `pokemon`, `onepiece`. Trade lists can contain cards from multiple games.
    </ResponseField>

    <ResponseField name="quantity" type="integer">
      The number of copies the user is willing to trade.
    </ResponseField>

    <ResponseField name="card_data" type="object">
      Normalized card metadata and current pricing from the Binder card catalog.

      <Expandable title="card_data">
        <ResponseField name="name" type="string">
          The card's display name.
        </ResponseField>

        <ResponseField name="set" type="string">
          The set or expansion code (e.g., `base1` for the Pokémon Base Set).
        </ResponseField>

        <ResponseField name="rarity" type="string">
          The card's rarity (e.g., `common`, `uncommon`, `rare`).
        </ResponseField>

        <ResponseField name="usd" type="string">
          Current non-foil market price in USD, as a decimal string. Use this field to estimate the value of a potential trade.
        </ResponseField>

        <ResponseField name="image_uri" type="string">
          A direct URL to the card's image, sourced from the game's card image provider.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Cases

<Warning>
  This endpoint returns `404 Not Found` if the share token does not exist or if the user has set their trade list to private. Both cases return the same status code — no information about the token's validity is disclosed.
</Warning>

| Status | Meaning |
|--------|---------|
| `200 OK` | The trade list is public and was found. The `holdings` array may be empty if no cards are marked for trade. |
| `404 Not Found` | No public trade list exists for the given token, or the trade list is private. |
