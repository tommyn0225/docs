Binder maintains a normalized card catalog spanning all supported trading card games. The Cards API gives you two endpoints: a **search endpoint** to query cards by name or game, and a **single-card lookup endpoint** that returns full card details along with a daily price history. Both endpoints are read-only, require no authentication, and return JSON.

---

## Search Cards

`GET /api/v1/cards` searches the catalog and returns a list of matching cards. Use this endpoint to build autocomplete fields, populate card pickers, or let users browse cards across games.

### Request

```http
GET https://yourdomain.com/api/v1/cards
```

### Query Parameters

<ParamField query="q" type="string">
  A text search string matched against card names. The search is case-insensitive and matches partial names — for example, `q=bolt` returns "Lightning Bolt", "Thunderbolt", and similar cards. Omit this parameter to retrieve cards without filtering by name.
</ParamField>

<ParamField query="game" type="string">
  Filter results to a single game. Accepted values are `mtg`, `pokemon`, and `onepiece`. Passing an unrecognized value returns a `400` error. Omit this parameter to search across all supported games simultaneously.
</ParamField>

<ParamField query="limit" type="integer">
  The maximum number of cards to return per request. Accepts values between `1` and `100`. Defaults to `50` if omitted.
</ParamField>

### Response

A successful request returns `200 OK` with a JSON object containing a `count` of cards returned, a `cards` array, and an `attribution` string.

```json
{
  "count": 1,
  "cards": [
    {
      "card_id": "e3285e6b-3e79-4d7c-bf96-d920f973b122",
      "name": "Lightning Bolt",
      "game": "mtg",
      "set_name": "Magic 2010",
      "set_code": "m10",
      "collector_number": "146",
      "rarity": "common",
      "image_url": "https://cards.scryfall.io/normal/front/e/3/e3285e6b-3e79-4d7c-bf96-d920f973b122.jpg",
      "price": {
        "usd": 1.50,
        "usd_foil": 3.00,
        "eur": 1.20
      }
    }
  ],
  "attribution": "Card data and pricing courtesy of Scryfall (MTG), PokéWallet (Pokémon), and OPTCG API (One Piece). Binder is an independent, non-commercial tool and exposes only its own value-added layer (binders, holdings, valuation, price history)."
}
```

#### Response Fields

<ResponseField name="count" type="integer">
  The number of cards returned in this response, after applying the `limit`. This reflects the size of the `cards` array.
</ResponseField>

<ResponseField name="cards" type="array">
  An array of matching card objects, sorted alphabetically by name, up to the requested `limit`. Returns an empty array if no cards match your query.

  <Expandable title="cards[]">
    <ResponseField name="card_id" type="string">
      The Binder-internal card identifier. For MTG cards, this is the card's Scryfall UUID. For other games, the format is game-specific.
    </ResponseField>

    <ResponseField name="name" type="string">
      The card's display name.
    </ResponseField>

    <ResponseField name="game" type="string">
      The trading card game this card belongs to: `mtg`, `pokemon`, or `onepiece`.
    </ResponseField>

    <ResponseField name="set_name" type="string">
      The full name of the set or expansion the card belongs to (e.g., `Magic 2010`).
    </ResponseField>

    <ResponseField name="set_code" type="string">
      The short code for the set or expansion (e.g., `m10` for Magic 2010, `base1` for Pokémon Base Set).
    </ResponseField>

    <ResponseField name="collector_number" type="string">
      The card's collector number within its set.
    </ResponseField>

    <ResponseField name="rarity" type="string">
      The card's rarity (e.g., `common`, `uncommon`, `rare`). May be `null` if rarity data is unavailable.
    </ResponseField>

    <ResponseField name="image_url" type="string">
      A direct URL to the card's image, sourced from the game's card image provider. May be `null` if no image is available.
    </ResponseField>

    <ResponseField name="price" type="object">
      Current pricing for this card.

      <Expandable title="price">
        <ResponseField name="usd" type="number">
          Current non-foil market price in USD. May be `null` if pricing data is unavailable.
        </ResponseField>

        <ResponseField name="usd_foil" type="number">
          Current foil market price in USD. May be `null` if no foil version exists or pricing is unavailable.
        </ResponseField>

        <ResponseField name="eur" type="number">
          Current non-foil market price in EUR. May be `null` if EUR pricing is unavailable for this card.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="attribution" type="string">
  A required attribution string for upstream card data providers. Include this text when displaying card data in your application.
</ResponseField>

<Note>
  The Cards API does not currently support offset or cursor-based pagination. If the catalog contains more results than your `limit`, narrow your search using the `q` or `game` parameters.
</Note>

## Fetch a Single Card

`GET /api/v1/cards/{game}/{id}` returns the full details for one card, including its daily price history. Use this endpoint to render a card detail view, display a price chart, or look up current pricing for a specific printing.

### Request

```http
GET https://yourdomain.com/api/v1/cards/{game}/{id}
```

### Path Parameters

<ParamField path="game" type="string" required>
  The trading card game the card belongs to. Must be one of `mtg`, `pokemon`, or `onepiece`. This value is case-sensitive.
</ParamField>

<ParamField path="id" type="string" required>
  The card's game-specific identifier. For MTG, this is the Scryfall UUID (e.g., `e3285e6b-3e79-4d7c-bf96-d920f973b122`). For Pokémon, it's a compact identifier such as `xy1-25`. You can obtain the correct `id` for any card from the [Search Cards](#search-cards) endpoint.
</ParamField>

### Response

A successful request returns `200 OK` with a JSON object containing a `card` object with the card's full metadata, a `history` array of daily USD price snapshots sorted chronologically from oldest to newest, and an `attribution` string.

```json
{
  "card": {
    "card_id": "xy1-25",
    "name": "Charizard",
    "game": "pokemon",
    "set_name": "XY Base Set",
    "set_code": "xy1",
    "collector_number": "25",
    "rarity": "rare",
    "image_url": "https://images.pokewallet.io/xy1-25.jpg",
    "price": {
      "usd": 300.00,
      "usd_foil": null,
      "eur": 275.00
    }
  },
  "history": [
    { "day": "2024-01-13", "price_usd": 290.00 },
    { "day": "2024-01-14", "price_usd": 295.00 },
    { "day": "2024-01-15", "price_usd": 300.00 }
  ],
  "attribution": "Card data and pricing courtesy of Scryfall (MTG), PokéWallet (Pokémon), and OPTCG API (One Piece). Binder is an independent, non-commercial tool and exposes only its own value-added layer (binders, holdings, valuation, price history)."
}
```

#### Response Fields

<ResponseField name="card" type="object">
  The card's full metadata. See the [Search Cards](#search-cards) response fields above for descriptions of each field — the `card` object here has the same shape as an item in the `cards` array.
</ResponseField>

<ResponseField name="history" type="array">
  A chronologically sorted array of daily USD price snapshots. Each element represents one day's recorded non-foil price. Only days with an available price are included — days with no recorded price are omitted.

  <Expandable title="history[]">
    <ResponseField name="day" type="string">
      The calendar date of the price snapshot in `YYYY-MM-DD` format. At most one entry exists per day.
    </ResponseField>

    <ResponseField name="price_usd" type="number">
      The non-foil USD market price recorded on that date.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="attribution" type="string">
  A required attribution string for upstream card data providers. Include this text when displaying card data in your application.
</ResponseField>

## Error Cases

<Warning>
  Both card endpoints return `404 Not Found` if the specified card does not exist in the Binder catalog. Verify that the `game` and `id` values you're using are correct — a mismatched `id` for the given game will result in a `404`. Passing an unrecognized `game` value to the search endpoint returns `400`.
</Warning>

| Status | Meaning |
|--------|---------|
| `200 OK` | The card was found and returned successfully. |
| `400 Bad Request` | An unrecognized `game` value was provided to the search endpoint. Expected `mtg`, `pokemon`, or `onepiece`. |
| `404 Not Found` | No card exists in the catalog for the given `game` and `id` combination. |
